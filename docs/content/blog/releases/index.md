# Release Notes

Use this section for shipped changes: new versions, notable fixes, migrations, deprecations, and release-ready highlights.

## Writing Checklist

- Put new posts in `docs/content/blog/releases/`.
- Prefer one release, one shipped milestone, or one focused launch story per post.
- Lead with what changed, who it affects, and whether readers need to take any action.
- Link to relevant docs pages, pull requests, demos, or migration instructions when possible.
